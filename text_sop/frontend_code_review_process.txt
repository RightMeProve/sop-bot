SOP: Frontend Code Review Process

Purpose:
This SOP outlines the step-by-step process for frontend code review process.

Scope:
This document applies to all relevant stakeholders involved in frontend code review process.

Responsibilities:
Relevant team leads and team members are responsible for executing this SOP as defined.

Procedure:
1. Ensure all code is pushed and committed to a feature branch.
2. Assign at least one peer reviewer with frontend experience.
3. Check for consistent use of HTML/CSS standards.
4. Verify responsive design works on different screen sizes.
5. Ensure accessibility standards (WCAG) are met.
6. Review for clean, reusable components.
7. Avoid inline styles and redundant code.
8. Check performance impacts of changes (e.g., large images, extra JS).
9. Add comments or suggestions directly on the pull request.
10. Approve or request changes and merge after all concerns are addressed.

References / Notes:
Please refer to internal documentation and tool-specific manuals for additional guidance.

--- End of SOP ---
